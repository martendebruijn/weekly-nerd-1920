Semantische HTML en ARIA

Semantische HTML en Accessible Rich Internet Applications (ARIA) zijn twee manieren om web applicaties toegankelijk te maken voor iedereen. Beide voegen betekenissen toe aan de inhoud van een web applicatie. Betekenissen die browsers, zoekmachines, schermlezers, RSS-lezers en uiteindelijk gebruikers de inhoud laten begrijpen.

Semantische HTML wordt vaak nog wel gebruikt, ARIA aan de andere kant niet. Waarom is dat? Dat vroeg Adam Silver, interactie designer en auteur van design- en CSS-boeken, zich ook af. Hij stelde de vraag op Twitter. Het gros gaf aan dat men zich er simpelweg niet van bewust zijn van het bestaan en van de voordelen van het gebruik van semantische HTML en van ARIA.

Why do people opt not to use semantic HTML?

Semantische HTML

Het <body> element van een HTML document bevat wat de gebruiker ziet op een pagina. Wanneer men hier tekst inzet, zonder aanvullende elementen, kan een browser geen onderscheid maken tussen de verschillende soorten inhoud.

See the Pen No markup by Marten de Bruijn (@martendebruijn) on CodePen.

Als een browser geen onderscheid kan maken tussen de verschillende soorten inhoud, kan hij deze inhoud niet op een betekenisvolle manier aan de gebruiker presenteren. Dit betekent ook dat losse elementen, zoals alinea’s en koppen, niet anders gestijld kunnen worden, het lastiger is voor zoekmachines om de inhoud te interpreteren en dat assisterende technologieën geen onderscheid kunnen maken tussen de verschillende elementen.

De tekst uit bovenstaand voorbeeld kan men visueel stijlen door ze in <div> elementen en in <span> elementen te zetten. Met CSS kan men deze elementen selecteren en stijlen.

See the Pen NWxwoGW by Marten de Bruijn (@martendebruijn) on CodePen.

Een <div> en een <span> element zijn echter niet semantisch. Ondanks dat men op deze manier de verschillende elementen apart kan stijlen, kan bijvoorbeeld een schermlezer nog steeds geen onderscheid maken tussen de verschillende elementen. Dit is waar semantische HTML om de hoek komt kijken. In HTML heeft men zes header elementen die aangeven dat het een koptekst is, <h1> tot <h6>. Voor alinea teksten bestaat het paragraph element, <p>. Wanneer deze elementen gebruikt worden, kan een schermlezer onderscheidt maken tussen de verschillende elementen.

See the Pen XWXzOXR by Marten de Bruijn (@martendebruijn) on CodePen.

Wanneer we het voorbeeld op deze manier opmaken, wordt er aan ondersteunende technologieën doorgegeven welke tekst een kop is en wanneer het om een alinea gaat. Ook kunnen zoekmachines en RSS-lezers nu wel onderscheidt maken. Hierdoor kan men ook een web pagina openen in leesmodes als een browser dit ondersteund, bijvoorbeeld Safari:

Het gebruik van semantische HTML is ook een voordeel voor developers. Bovendien is het minder code omdat men elementen nu geen klassen hoeft te geven, het is ook leesbaarder voor andere developers die jouw code lezen. Ze zien meteen wat voor een element iets is.

Semantische HTML voor specifieke doelen zorgen ook voor relevant gedrag van een browser. Neem het element <input rel="tel">. Dit element is een <input> element: het verteld een browser dat de gebruiker hier iets kan invoeren. Het attribuut type=„tel“ vertelt de browser dat het hier een telefoonnummer verwacht. Wanneer een gebruiker op zijn mobiel op dit input element klikt krijgt men een speciaal toetsenbord, ontworpen om telefoonnummers in te voeren, te zien.

ARIA

Semantische HTML is dus een goed idee om te gebruiken, maar wat is ARIA? ARIA is ontworpen om web applicaties toegankelijker te maken voor ondersteunende technologieën. Stel je hebt een formulier waar men zijn e-mailadres moet invullen. Wanneer de invoer om welke reden dan ook niet toegestaan is, kan men een foutmelding laten zien op het scherm. Dit kan men doen door bijvoorbeeld <span>Voer een geldig mail adres in, zoals voorbeeld@gmail.com.</span>. Het probleem hier, is dat deze foutmelding bijvoorbeeld niet voorgelezen zal worden door een schermlezer. Wanneer we het attribuut aria-describedby toevoegen aan het input element (en dus niet de foutmelding zelf!) en de foutmelding zelf een ID geven, gebeurd dit wel.

<span> id="email-error">Voer een geldig mail adres in, zoals voorbeeld@gmail.com.</span> <input type="email" aria-describedby="email-error">
ARIA en JavaScript

ARIA komt pas echt tot zijn recht wanneer JavaScript gebruikt wordt. Sterker nog JavaScript is meestal noodzakelijk om ARIA goed te laten functioneren. Stel we hebben een drop-down menu. Met het attribuut aria-expanded kunnen we ondersteunende technologieën laten weten of het menu open is of niet. Wanneer het menu niet open is, is aria-expanded="false". Als het menu open is, is aria-expanded="true". In HTML kan men de standaard waarde meegeven. De standaard waarde is de waarde die het moet zijn wanneer de pagina geladen wordt. Als het menu gesloten moet zijn wanneer de pagina laadt, dan moet men dus aria-expanded="false" definiëren. Wanneer de gebruiker het menu opent, wordt dit attribuut niet vanzelf aangepast. Dit moet men m.g.v. JavaScript veranderen naar true.

Eerste regel van ARIA

Men kan ARIA gebruiken om niet-semantische elementen toegankelijker te maken. Het is echter niet de bedoeling dat men ARIA gebruikt i.p.v. semantische elementen. Dit resulteert in meer code en het zal alleen maar werken voor gebruikers die technologieën gebruiken die de ARIA attributen ook daadwerkelijk begrijpen. De eerste regel van ARIA luidt dan ook: gebruik het niet als het niet nodig is. Als je een HTML element of attribuut kan gebruiken waar de semantiek en het gedrag ingebouwd is i.p.v. een ARIA attribuut te gebruiken, doe dat dan ook.

Conclusie

Semantische HTML en ARIA zijn twee manieren om web applicatie toegankelijker te maken. Vooral ARIA wordt vaak niet niet gebruikt en begrepen. Dit komt omdat menig developer/designer zich er niet van bewust is.

Semantische HTML elementen, zoals <article>, geven betekenis mee aan de inhoud. Ondersteunende technologieën weten hierdoor met wat voor een soort inhoud zij te maken hebben en kunnen ze dit door communiceren naar de gebruiker.

Wanneer men geen gebruik kan maken van een semantisch element, kan men gebruik maken van ARIA. Het is vaak noodzakelijk om hier JavaScript aan te koppelen. ARIA is niet bedoeld om te gebruiken i.p.v. semantische elementen. De eerste regel is dan ook: gebruik het niet als het niet nodig is.
